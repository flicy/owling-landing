# AI 儿童陪伴硬件：国内外合规法案要点（2025-2026）

## 一、中国

### 《人工智能拟人化交互服务管理暂行办法》（2026年7月15日生效）

这是目前对 AI 陪伴产品最直接相关的法规。

| 要点 | 内容 |
|------|------|
| **适用范围** | 模拟人类特征（性格、思维、沟通方式）提供持续情感互动的 AI 服务 |
| **对未成年人的禁止** | **严禁**向未成年人提供"虚拟亲密关系"服务（虚拟家人、虚拟伴侣） |
| **家长同意** | 14岁以下使用任何拟人化交互，必须获得监护人同意 |
| **未成年人模式（强制）** | 必须设置"未成年人模式"，包括：使用时长限制、内容过滤、监护人监控机制、风险预警、消费限制 |
| **市场影响** | 多家大厂已下架或调整情感陪伴类 agent 功能 |

### 对灵偶项目的影响

- ✅ 定位"英语学习+成长陪伴"而非"虚拟亲密关系"，方向安全
- ⚠️ 必须有明确的家长同意机制（首次激活需家长确认）
- ⚠️ 必须有未成年人模式：时长控制、家长可查看/管理
- ⚠️ 对外表述避免"AI朋友""AI家人"等拟人化亲密关系暗示
- ✅ 家长端设计本身已满足"监护人监控"要求
- ⚠️ 不能有任何消费诱导设计

### 其他相关法规
- 《儿童个人信息网络保护规定》：收集14岁以下儿童信息需监护人明示同意
- 《生成式人工智能服务管理暂行办法》：生成内容不得危害未成年人身心健康
- 《未成年人保护法》：网络产品需设未成年人保护机制

## 二、美国

### COPPA 更新规则（2026年4月22日生效）

| 要点 | 内容 |
|------|------|
| **数据最小化** | 只收集服务必需的最少儿童数据 |
| **家长同意** | 13岁以下需可验证的家长同意（verifiable parental consent） |
| **AI 特别要求** | 明确覆盖 AI 系统收集和处理儿童数据的方式 |
| **透明度** | 必须清晰告知家长数据如何被使用 |

### 州级立法

| 州 | 法案 | 要点 |
|----|------|------|
| **马里兰州** | AI 玩具安全法案（2026.7.1 生效） | 上市前安全评估、严格隐私规则、禁止用数据营销、违规罚5万美元/次 |
| **加州等** | 多州提案 | 要求 AI 交互必须明确告知孩子"这不是真人"、检测自我伤害内容、禁止设计导致不健康情感依赖 |

### 对灵偶项目的影响（若出海北美）
- ⚠️ 必须实现可验证的家长同意流程
- ⚠️ 语音数据需端侧处理或明确告知云端使用
- ⚠️ 记忆功能需提供删除机制
- ⚠️ 需要在交互中让孩子知道对方是 AI

## 三、欧盟

### EU AI Act + 玩具安全法规

| 要点 | 内容 |
|------|------|
| **高风险分类** | 可能操纵儿童或鼓励危险行为的 AI 应用被禁止 |
| **数字产品护照** | 玩具需附带 QR 码可追溯安全信息 |
| **GDPR 儿童条款** | 儿童数据处理受最严格保护，各国年龄门槛 13-16 岁不等 |

### 对灵偶项目的影响（若出海欧洲）
- ⚠️ 需通过安全评估证明不会操纵或诱导儿童
- ⚠️ 隐私政策必须"儿童可理解"
- ⚠️ 家长需有完整数据访问和删除权

## 四、产品设计合规建议总结

| 设计要素 | 建议 |
|----------|------|
| **首次激活** | 强制家长扫码/验证身份后才能使用 |
| **时长控制** | 默认每日使用时长上限（如30分钟），家长可调 |
| **内容过滤** | 玩偶输出经过安全层过滤，不生成暴力/恐怖/成人内容 |
| **数据存储** | 语音本地处理优先，上云需明确告知并获授权 |
| **记忆管理** | 家长可查看、编辑、删除所有记忆内容 |
| **消费设计** | 不得有任何诱导充值、抽卡、掉血等机制 |
| **角色定位** | 对外说"学习伙伴"而非"AI朋友/AI家人"，避免拟人化亲密关系定性 |
| **AI 身份披露** | 交互中需让孩子理解这是 AI 而非真人 |
| **情绪处理** | 检测负面情绪后引导孩子与家长沟通，不替代心理咨询 |
| **退出机制** | 物理按钮或明确的关闭方式，而非只能软件操作 |

## 关键判断

**灵偶项目定位在"英语学习+成长记录+家长可见"，核心安全。** 风险点主要在：
1. 拟人化程度的表述边界——叫"朋友"还是叫"学习伙伴"
2. 记忆功能的数据合规——本地部署是加分项
3. 使用时长和家长控制——必须有，不能缺
