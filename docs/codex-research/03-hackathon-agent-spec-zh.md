# 黑客松执行规格

## Demo 目标

在 3-5 分钟内让观众理解：这是一个能记住孩子、主动陪孩子说话、和孩子一起成长的 AI 陪伴玩偶原型。

## 推荐 Demo 剧本

1. 孩子第一次唤醒玩偶，告诉它自己的名字、喜欢的东西和今天想学的英语主题。
2. 玩偶用温柔、儿童友好的语气回应，并邀请孩子讲今天发生的一件事。
3. 孩子说一个简单事件，比如今天画了一只蓝色恐龙、和爸爸吵架、想学怎么说“我有点紧张”。
4. 玩偶把这个事件变成“今天的记忆叶子”或“共同故事树”的一部分。
5. 第二轮对话中，玩偶主动引用这段记忆：比如“你昨天画的蓝色恐龙今天想学一个新单词吗？”
6. 家长端或展示面板出现摘要：孩子今天的兴趣、情绪、英语表达、可继续聊的话题。

## MVP 功能清单

- 语音输入：可用浏览器录音、移动端录音或硬件按钮模拟。
- 语音输出：TTS 或预设音色均可，重点是连贯体验。
- 对话 agent：能主动提问、回应孩子、保持儿童友好语气。
- 记忆存储：本地 JSON / SQLite / 简单数据库均可。
- 记忆调用：下一轮对话必须能引用上一轮或上一次 session 的信息。
- 成长可视化：记忆叶子、故事树、伙伴等级、兴趣地图任选一种。
- 家长摘要：至少展示文本摘要，不必做完整家长 App。

## 玩偶人格建议

- 主动但不过度打扰。
- 像朋友，也像家庭里的温柔记录者。
- 不替代父母，不扮演成人亲密关系。
- 遇到负面情绪时先接住，再鼓励孩子和家长沟通。
- 对英语学习保持轻量、游戏化、低压力。

## 可视化建议

优先选择一种隐喻，不要同时做太多：

- 记忆叶子：每天对话生成一片叶子。
- 共同故事树：孩子和玩偶一起写的故事慢慢长成树。
- 伙伴等级：玩偶随着陪伴次数升级，但避免强诱导充值。
- 兴趣地图：展示孩子最近反复提到的主题。

## 黑客松不要做的事

- 不要做完整儿童社交平台。
- 不要做复杂多 agent 战斗、充值、掉血系统。
- 不要做真实硬件量产方案。
- 不要把大部分时间花在外壳和 PCB 上，除非硬件已经现成。
- 不要让产品变成单纯英语刷题工具。
- 不要在儿童场景里引入不可控开放聊天社区。

## 评委能听懂的差异点

普通 AI 玩具回答孩子的问题；这个玩偶会记住孩子的生活，并和孩子共同成长。

普通英语学习产品强调练习效率；这个产品把英语放进孩子自己的故事、情绪和家庭关系里。

普通硬件是一个终端；这个玩偶是孩子、父母、实体玩偶和云端 agent 之间的关系入口。
